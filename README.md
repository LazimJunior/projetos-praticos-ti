## 🗂️ O que construí nesses 30 dias?

Dividi os 12 projetos em semanas temáticas para mostrar como a TI se conecta de ponta a ponta.

### 🟢 Semana 1: Organizando o Caos (Automação e Dados)
*Nesta fase, o foco foi transformar processos manuais lentos em fluxos automáticos e confiáveis.*
* `01-limpeza-dados-chamados/`: Script Python que "faxina" planilhas de suporte automaticamente.
* `02-extracao-sla-sql/`: Queries inteligentes para descobrir onde o suporte está perdendo tempo.
* `03-automacao-onboarding/`: Python cuidando das boas-vindas: criação de acessos num clique.

### 🔵 Semana 2: A Visão de Negócios (Inteligência e Nuvem)
*Aqui, transformei dados brutos em decisões visuais e levei a infraestrutura para o próximo nível.*
* `04-dashboard-performance-ti/`: Painel no Power BI para que gestores entendam a saúde da operação.
* `05-infraestrutura-aws/`: Provisionamento de nuvem usando código (IaC) — rápido e seguro.
* `06-dashboard-custos-cloud/`: Onde está indo o orçamento? Monitoramento de custos na AWS.

### 🟠 Semana 3: A Ponte do Conhecimento (Governança)
*TI não é só máquina; é clareza. Usei as boas práticas da ITIL para organizar a casa.*
* `07-one-pager-suporte/`: Um guia visual "vapt-vupt" para o usuário resolver o básico sozinho.
* `08-matriz-escalonamento/`: O mapa do caminho — quem resolve o quê e quando escalar?
* `09-documentacao-scripts/`: Porque código sem documentação é um problema para o futuro.

### 🟡 Semana 4: O Fator Humano (Comunicação e Didática)
*Fechando o ciclo com o que mais importa: as pessoas que usam a tecnologia.*
* `10-guia-pacote-office`: Dicas de produtividade pensadas para quem não é "da área".
* `11-checklist-chamados-ti/`: Ensinando o usuário a pedir ajuda de forma eficiente.
* `12-portfolio-integrado`: O nó que amarra todas as soluções deste repositório.
